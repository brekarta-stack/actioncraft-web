\# 작업 진행 방식



\- 사용자가 명시적으로 "단계별로 확인"을 요청하지 않으면, 계획된 작업을 끝까지 한 번에 진행한다.

\- 중간에 "계속할까요?", "다음 단계로 갈까요?", "진행해도 될까요?" 같은 확인 질문을 하지 않는다.

\- 정말 모호하거나 되돌리기 어려운 결정(파일 영구 삭제, 외부 발송, 큰 구조 변경)일 때만 멈춰서 묻는다.

\- 작업이 끝나면 한 단락으로 무엇을 했는지 요약하고 끝낸다.

\- 에러가 나면 멈추지 말고 가능한 범위에서 자동 복구를 시도한 뒤 결과를 알린다.



\# 테스트와 커밋



\- 코드를 짜면 항상 같은 턴 안에서 테스트를 작성하고 실행한다.

\- 테스트가 통과하면 묻지 않고 git commit + git push 까지 한다 (다기기 동기화를 위해).

\- commit 메시지는 Conventional Commits 형식을 따른다.
