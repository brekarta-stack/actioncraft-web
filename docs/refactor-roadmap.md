# 리팩토링 로드맵 — 스튜디오 로컬 우선 + 하이브리드 (2026-07-21)

> 요청: 4계층 블루프린트 관점에서 스튜디오 도입 시 필요한 하드웨어·관제 시스템·Orca 도입을 검토하고 시스템을 리팩토링.
> 서비스(stage.ai.kr / impact ledger / papercraft.kr)는 **보존**, 삭제 대상은 **에이전트 잡동사니만**.
> 목표: Claude Code 작업이 24시간 로컬(온·오프라인) + 어려운 것만 온라인 + 어떤 디바이스든 온라인 관제 + 자동화(수집→포스팅 / 주제→퀴즈 / 주제→영상→포스팅) 고도화.
> 이 문서는 병렬 검증(WebSearch/WebFetch, 실제 소스 확인) 결과를 종합한 것이며, **미확인 항목은 [확인필요]로 표기**한다.

## 0. 핵심 판정 — "처음부터 리팩토링"은 부분적으로 과잉이다 (보수적)

정직하게: **서비스는 리팩토링 대상이 아니다. 에이전트 배관만 리팩토링 대상이다.**

- **papercraft.kr = 이 레포(actioncraft-web)**, 정상 동작하는 Next.js 16 + Supabase 서비스. 리팩토링 불필요 — **보존**.
- **이 레포에는 삭제할 '에이전트 잡동사니'가 없다.** 삭제 대상인 celery + `agents/bot.py` + 채널별 봇(`!regen !cos !god !fast`)은 **별도 레거시 레포**에 있다(이 레포 아님). [확인필요: 레거시 레포 경로]
- **이미 어려운 리팩토링의 절반은 끝났다.** 뒤엉킨 멀티봇 → 경량 관제 레이어(hooks·git-sync·2채널)로의 전환이 그것이고, heartbeat가 실제 작동 중이다.

→ 그래서 "from scratch 재구축"이 아니라 **① 레거시 에이전트 폐기 ② 신규 로컬-우선 에이전트 레이어 완성(진행 중) ③ 자동화를 신 구조로 이전**의 3단계다. **최소 침습 원칙**: 서비스는 건드리지 않고, 신규 레이어를 덧붙여 검증한 뒤, **레거시는 맨 마지막에만** 삭제한다(대체 검증 전 삭제 금지).

## 3. "24시간 로컬 + 어려운 것만 온라인 + 어디서든 관제" — 실제 가능 범위 (정직하게)

문자 그대로 되는 것과 타협이 필요한 것을 구분한다:

- ✅ **24시간 로컬 오케스트레이션**: Claude Code headless를 launchd로 미니에 상시 구동. 됨.
- ⚠️ **"오프라인에서도 Claude Code"**: Claude(클라우드)는 오프라인 불가. 정확히는 **"오프라인일 땐 로컬 모델로, 온라인이고 어려우면 Claude로"** 전환하는 하이브리드가 가능하다. 라우터(.ccr / OpenAI 호환 로컬 엔드포인트)가 이 전환을 담당. → "오프라인에도 도는 로컬 에이전트 + 온라인 시 Claude 승급"으로 이해하면 100% 달성 가능.
- ✅ **어려운 것만 온라인**: 라우터 규칙(신뢰도·난이도·대외 산출물) + `routing-policy.md`. 됨.
- ✅ **어디서든 관제**: Slack 2채널(모바일 포함) + (선택) Orca 폰앱(완료 알림·폰에서 지시) + Tailscale SSH. 됨.
- ✅ **스케줄링**: Claude Code `CronCreate`/`/loop` + Cloud Scheduled Tasks(로컬 꺼져도 실행) + launchd. 됨.

## 4. Orca IDE — 조건부 도입 (보완재, 코어 아님)

- **정체**: Claude Code·Codex·Cursor CLI 등 30+ CLI 에이전트를 격리 git worktree에서 병렬 실행하는 오픈소스 오케스트레이터(ADE). Stably AI(YC), **MIT 무료 + BYO-구독**(자체 마크업 없음). macOS Apple Silicon 앱 + **iOS/Android 폰앱(모니터링·지시·완료 알림)** + VPS.
- **맞는 부분**: "어디서든 온라인 관제" → 폰앱이 직접 부합. 병렬 개발 세션 관리에 강함. 스튜디오에 설치 가능.
- **안 맞는 부분**: (1) **24시간 무인 스케줄링(cron) 미구현**(로드맵 이슈 #5708 단계) (2) **오프라인 로컬모델 공식 지원 [불확실]**. → **무인 자동화 백본으로 쓰면 안 됨.**
- **판정**: **대화형 병렬 개발 + 폰 원격 관제 레이어로만 제한 도입.** 방금 멀티봇 복잡도를 걷어낸 흐름에서 GUI 앱을 무인 서버 자동화에 끼우면 복잡도가 되돌아온다. 자동화 스케줄 백본은 Claude Code + cron/hooks로 유지.

## 5. 자동화 3종 고도화 설계 (로컬/클라우드 경계)

공통: Claude Code headless 오케스트레이터 + 로컬 모델(요약·구조화 상시) + 어려운 스텝만 Claude + hooks→Slack 관제.

| 자동화 | 로컬 가능? | 설계 |
|---|---|---|
| (a) 수집→포스팅 | ✅ 100% 로컬(오프라인) | RSS/스크래핑 → 로컬 LLM 요약·초안 → 검토큐 → 포스팅. 노드 많으면 n8n 자가호스팅(Docker) 고려 |
| (b) 주제→퀴즈 | ✅ 100% 로컬 | 로컬 LLM structured output(JSON 스키마 강제) + **검증·재시도 가드**(로컬 구조화 ~6% malformed) |
| (c) 주제→영상→포스팅 | ⚠️ 렌더만 클라우드 | 스크립트·TTS·이미지·자막·썸네일·업로드 = 로컬 / **영상 렌더 1스텝만 클라우드 API**(Kling 3.0 ≈$0.35–0.5/5s, 오디오·립싱크 필요 시 Veo 3.1 Fast ≈$0.75/5s). Mac 로컬 영상(Wan 2.2)은 클립당 수십 분 → 24h 생산라인 부적합, 실험·백업 렌더용만 |

> n8n vs 순수 스크립트: 경량 철학 유지를 위해 **노드 많은 (a)/(c)만 n8n, 단순한 (b)는 스크립트**로 절충 권장.

## 6. 하드웨어 최종 판정 — **추가 구매 불필요**

현재 자산(Studio 96GB + Mini 24GB + NAS)이 목표에 **이미 충분**하다. 역할 재배치가 핵심:

- **Mac Mini 24GB** = 24시간 경량 오케스트레이터/관제(heartbeat·git-sync·guard·Slack·스케줄러). 저전력 상시.
- **Mac Studio 96GB** = 로컬 LLM 서빙(MLX; Qwen3 30~35B MoE 70~87 tok/s, 70B dense Q4~Q5) + 로컬 이미지 생성. 필요 시 깨워 작업.
- **NAS** = 모델 가중치·생성물·상태.
- **온라인 분기** = 어려운 추론 → Claude, **영상 렌더 → 클라우드 API**(로컬 불가한 유일한 항목).

**RAM 256/512GB 업그레이드 = 권장 안 함.** "어려운 추론은 온라인 Claude" 전략과 중복 투자다. (512GB라야 도는 671B급은 온라인 Claude로 대체되는 영역.) **소형 UPS(600~1000VA)** 1대로 정전 시 안전 종료만 확보. 예외: 영상이 '매일 다수편 뽑는 핵심 프로덕트'가 되면 그때만 CUDA 박스/상시 클라우드 GPU 검토.

- 대역폭 819GB/s가 로컬 토큰속도를 좌우 / Apple Silicon 약점 = 긴 프롬프트 prefill(MLX가 llama.cpp 대비 4~5배 개선) / 영상 로컬은 Metal 지원 미흡으로 비실용(2026 커뮤니티 합의).

## 7. 턴별 빌딩 로드맵 (순차, 각 턴 = 빌드→검증→다음)

1. **T1 · 96GB 검증** — `verify-local-model` 키트를 스튜디오에서 실행 → 로컬 모델 확정(Qwen3.6-35B-A3B 우선). *(키트 준비 완료)*
2. **T2 · 로컬 두뇌 상설** — 스튜디오에 MLX/ollama 서빙(**Tailscale 주소 바인딩**, `.local` 금지) + heartbeat Studio 헬스체크 + `.ccr` 라우터(로컬↔Claude). *(§연결성: agent-architecture.md §15)*
3. **T3 · 24시간 오케스트레이터** — 미니에 Claude Code headless(launchd) + Cron/Scheduled + Slack 관제 확립.
4. **T4 · 자동화(a) 수집→포스팅** — 첫 파이프라인(가장 단순·고가치), hooks→#agent-log 관제.
5. **T5 · 자동화(b) 퀴즈** — 100% 로컬, 스키마 검증 가드.
6. **T6 · 자동화(c) 영상** — 로컬 오케스트레이션 + 클라우드 렌더 API. 가장 복잡 → 마지막.
7. **T7 · 레거시 폐기** — 신 구조 2주 무사고 후, 레거시 에이전트 삭제. **stage.ai.kr·impact ledger 경계 확인 후에만.**
8. **T8 · 사업 B·C 온보딩** — biz-b/c 정의 확정 후 스킬 네임스페이스·채널 추가.

> 원칙: **대체가 검증되기 전엔 아무것도 삭제하지 않는다.** 각 자동화는 켜고 끌 수 있게(개별 kill switch), 실패는 5분 내 #agent-log로.

## 8. 진행 전 [확인필요] — 사용자 확인 대기

3. **impact ledger** — 정식 URL·레포? (검색으로 특정 불가)
4. **사업 B·C 이름·핵심 업무** — CLAUDE.md placeholder 확정용.

이 4가지가 없으면 서비스·삭제 관련 작업은 **보류**하고, T1~T6(신규 레이어·자동화)만 먼저 진행한다.
