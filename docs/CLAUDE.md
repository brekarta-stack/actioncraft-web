# CLAUDE.md — 개인 에이전트 오케스트레이션 시스템 설계 마스터 컨텍스트

> **Claude Code에게**: 이 파일은 이 프로젝트의 상시 컨텍스트다. 세션 시작 시 이 문서를 기준으로 삼아라.
> **너의 임무**: 아래 확정 사항의 *근거를 존중하되 맹신하지 말고*, 사용자의 기존 프로젝트 코드/맥락과 대조해 **적대적으로 재검토**한 뒤, §D의 "비어 있는 결정"을 채워 더 나은 설계도와 구현 계획을 산출하라.
> **불변 원칙**: (1) 클로드 편향 금지 — 멀티모델 전제. (2) 2주 완성·바이브코딩 초보 제약 최우선. (3) 확정을 뒤집으려면 반드시 근거 제시. (4) 1인 시스템의 적은 부족한 확장성이 아니라 과잉 복잡성 — 단순함·복구가능성·관측성 우선.

## A. 고정 전제 (변경 불가)

- 1인 운영, 초보, **총 2주** 완성.
- 하드웨어: **Mac Studio M3 Ultra 96GB** / **Mac mini M4 24GB** / **UGREEN DXP4800 Plus NAS**(x86, RAM 8→32GB 증설).
- **탈클로드**: Claude 외 OpenAI/Kimi/로컬(Ollama) 등 작업별 엔진 선택.
- **무중단(최우선)**: 사용자가 Claude Code 구독 토큰을 다 써도 운영·관제 지속 → 운영 런타임은 **API 키/로컬 기반**, Claude Code는 개발 도구로만.
- **토큰/비용 최적화**가 핵심 가치.

## B. 운영 목적 10 + 1주차 필수

주식 자동매매(KR+US), 자료수집·학습(퀴즈·아카이빙), 링크드인 자동발행, 업무+구글일정, 사업 파트별 리드발굴+블로그+견적서(**파트 대개편 예정**), 멀티모델 라우팅, 영상 파이프라인, 프로젝트관리+관제(편리한 스킬편집), 무중단·토큰최적화, 보이스 어학(영/일).
**1주차 필수 4종**: 업무관리 · 사업 자동화 · 포스팅 발행 · 관제.

## C. 확정 아키텍처 (근거 포함, 재검토 대상)

**C-1 코어 패턴** — n8n(NAS)이 결정적 워크플로로 실행, LLM은 워크플로 안 "한 스텝"으로만. *채택 근거*: 반복업무 결정성·토큰절감·관측성·인젝션 표면 축소. **폐기**: 채널별 상시 챗 에이전트가 실행 주체가 되는 구조. 슬랙은 폐기 아님 → "보고·승인 창구"로 축소(#ops #approvals #trading #inbox).

**C-2 모델 관문** — LiteLLM(NAS) 단일 게이트. 작업별 라우팅·비용집계·예산캡·폴백(프리미엄→Kimi→로컬). 워크플로는 실제 모델명이 아닌 **역할 별칭**만 호출. 무중단의 실체 = 예산소진 시 자동 강등.

**C-3 하드웨어 역할** — NAS(24/7 백본: n8n+Postgres, LiteLLM, Qdrant, 백업데몬), Studio(온디맨드 추론: 코딩·한국어·요약·퀴즈·보이스·로컬이미지, JIT 로드), mini(24/7 상시: Qwen9B+BGE-M3, **Uptime Kuma 관제는 여기** — 감시자·감시대상 분리).

**C-4 관제** — n8n Insights + LiteLLM UI + Uptime Kuma + 슬랙 알림 + 정적 허브 1p. 자체 대시보드 재제작 금지. "편리한 스킬 편집" = n8n 웹 편집기.

**C-5 매매(타협 불가)** — LLM 분석가(장전/장후, 읽기전용, 주문권한 없음)가 구조화 JSON 제안 → 결정적 엔진(python-kis)이 하드 가드레일 검증 후 주문. 상태머신·킬스위치·일손실한도·레이트리밋(20/s 토큰버킷)·토큰 세션앵커 갱신·웹소켓 재접속. 브로커=**KIS OpenAPI**(국내+미국, 모의투자, 순수 REST/WS). 토스는 2차/실험용. 본인계좌 자동매매만 합법(타인자금=무등록 투자일임·불법).

**C-6 보안·네트워크** — Tailscale 사설망(관리 UI 전용), 슬랙 Socket Mode(인바운드 0), **NAS 포트포워딩 금지**. 최소권한(매매 LLM 읽기전용 키). **IP 안정화(필수)**: 랜선 재연결·재부팅 시 DHCP로 로컬 IP가 바뀌어 기기 간 연결이 끊길 수 있음 → (1) **기기 간 참조는 IP 하드코딩 금지, Tailscale MagicDNS 호스트네임으로만**(물리 랜과 무관하게 고정), (2) 공유기에서 **NAS·맥 DHCP 예약(고정 IP)** 설정, (3) 같은 NAS 내 컨테이너끼리는 컨테이너 이름으로 통신(IP 무관). **물리 단절 주의**: NAS 랜선을 자주 뽑으면 24/7 백본 전제가 깨짐 → NAS 랜선은 상시 연결 권장, 불가피 시 단절 구간엔 매매 등 실시간 작업 미실행 + Uptime Kuma로 "NAS 연결 끊김" 감지.

**C-7 모델 로스터 + 라우팅** — 상세는 `routing-design.md`. 요지: 로컬 Qwen3-Coder-30B(상시)/Next(JIT), 한국어 Qwen3.6/A.X(Apache2.0), mini Qwen9B+BGE-M3; 클라우드 Claude/GPT/Kimi. 별칭 11종 + 폴백 + 4대 게이트(자동승급·예산·프라이버시·시간대). EXAONE는 비상업 라이선스 → 개인학습 전용.

## D. 비어 있는 결정 (Claude Code가 채울 것) — 상세는 `design-maturity-checklist.md`

**착수 전 반드시 닫을 3개(게이팅)**:
1. **매매 엔진 위치**: NAS(백본 일관성) vs mini(장중 웹소켓 근접) — 트레이드오프 판단.
2. **멱등성 구현**: 매매 주문(주문ID)·발행(콘텐츠 해시)·이메일(발송키) 중복 실행 방지. 재시도해도 부작용 1회.
3. **RTO/RPO 목표 + LiteLLM 장애 시 매매 LLM 직접 폴백 경로**.

**그 외 열린 결정**: 데이터 스키마·SSOT 정의(리드/파트/매매), 재시도·백오프·DLQ 정책, 키 로테이션, 프롬프트 인젝션 신뢰경계, IaC를 Git으로·복원 리허설·롤백 절차, 별칭별 실제 예산금액, 기존 40잡 이주 순서(교살자, "켜기 전에 끈다" 중복방지).

## E. 안티패턴 (반복 금지)
LLM 직접 주문권한 부여 금지 · 자체 관제 대시보드 재제작 금지 · 채널별 상시 챗 에이전트로 실행백본 구성 금지 · 운영을 Claude Code 구독토큰에 의존 금지 · 네이버 블로그 셀레니움 자동발행 금지(계정정지 리스크) · Kubernetes/마이크로서비스 과분할/다중리전 등 1인 규모 과잉설계 금지.

## F. 사실 참고 (재검토 시 유용, 2026-07 리서치 기준)
- 96GB로 Qwen3-Coder-30B(상시)+Next(JIT) 실사용 가능, 동시 상주는 불가(교대 로드).
- GLM-5.2 등 프론티어 오픈모델은 96GB 불가·512GB라도 느려서 비용대비 무의미 → 필요 시 클라우드 API. 로컬 가치는 "공짜·무제한·프라이버시"이지 프론티어 품질 아님.
- 네이버블로그/티스토리 공식 글쓰기 API 폐지 → 완전자동화는 WordPress, 네이버는 반자동.
- LinkedIn 심사(수주~수개월)·YouTube API 감사(2~4주, 미통과 시 업로드 비공개) → 0일차 신청.
- 백업: 실행중 Postgres 스냅샷 불충분 → pg_dump + N8N_ENCRYPTION_KEY 별도 보관 + 복원 리허설. 이미지 버전 태그 고정(n8n latest 금지).

## G. 2주 실행 계획 (요약)
- **0일차**: LinkedIn/YouTube/KIS 신청, 토스 사전신청, API키 발급, NAS RAM 증설, 모델 다운로드.
- **1주차**: (1~3)NAS 스택+LiteLLM 라우팅+로컬모델+Tailscale, (3)슬랙 4채널, (4)업무/일정, (5~6)사업자동화+견적PDF, (7)포스팅발행+관제 = **1주차 관문**.
- **2주차**: (8~9)학습·퀴즈, (10~13)트레이딩(모의투자 왕복 1건=성공), (12~13)보이스, (14)영상 E2E 1편. **컷 순서: 영상→보이스→퀴즈.** 트레이딩은 반쯤 만들 바엔 3주차로 미룰 것.

## H. 아키텍처 지식 체크리스트 (설계 시 11영역 누락 방지)
