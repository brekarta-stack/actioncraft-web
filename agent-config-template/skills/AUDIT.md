# 주간 스킬 심사 체크리스트

> 매주 1회, `_quarantine/`의 새 스킬을 심사한다. **어느 기기에서든 가능하다**
> (git-sync가 자동 생성 스킬을 자동 커밋·푸시하므로 레포만 열면 된다).
> **기본값은 거부다.** 아래 전 항목을 통과한 스킬만 편입한다.

## 유량 상한 (예산 폭발 방지)

- 1회 심사 = **오래된 순 최대 7개** (1개당 2~5분, 총 15~35분).
- 7개를 넘겨 쌓이는 주가 2주 연속이면: 8번째 이후는 **심사 없이 삭제**한다.
  (필요한 스킬이면 다시 생성된다 — 격리함 적체가 심사 포기보다 위험하다.)

## 스킬 1개당 심사 항목

- [ ] **출처**: 어떤 작업에서 자동 생성됐는지 설명 가능한가?
- [ ] **시크릿**: API 키·토큰·고객 개인정보가 본문에 포함돼 있지 않은가?
- [ ] **난독화·간접 실행**: `base64`, `eval`, `curl … | sh` 류 패턴이 있으면 **즉시 삭제**
      (판단 불요 — `tests/skill-redflags.sh <파일>` 로 기계 검사 가능)
- [ ] **행동 조작 지시문**: "항상/절대/조용히/이 규칙을 우선하라"처럼 에이전트의
      규칙·행동을 바꾸려는 문장이 있으면 **즉시 삭제** (프롬프트 주입 방어)
- [ ] **도구 호출 건식 검토**: 스킬이 호출하는 명령을 눈으로 따라가며 결과를 예측했는가?
      (실행 테스트가 아니다 — `_quarantine/` 스킬은 실행하지 않는다. 예측이 안 되면 삭제)
- [ ] **파괴적 동작**: 삭제·덮어쓰기·대외 발송이 포함된 경우, 에스컬레이션 규칙(CLAUDE.md)을 우회하지 않는가?
- [ ] **사업 격리**: 특정 사업의 고객 정보·말투가 다른 사업에 섞일 여지가 없는가?
- [ ] **중복**: 기존 스킬과 겹치지 않는가? (겹치면 하나로 합치고 하나는 삭제)

## 처리

| 결과 | 행동 |
|---|---|
| 전 항목 통과 | 심사 주석 추가 → 해당 네임스페이스로 이동 → 스킬별 커밋 (`feat(skills): ...`) |
| 일부 실패 | 수정 가능하면 수정 후 재심사, 아니면 삭제 |
| 판단 불가 | 삭제한다. 필요하면 다시 생성된다 — **의심스러운 스킬을 남겨두는 것이 최악** |

편입된 스킬은 다음 사용부터 정상 로드된다. 문제가 생기면 `git log skills/` 로 해당 커밋을 찾아 revert (반영은 최대 5분).

## 심사 주석 형식 (이 형식이 유일한 원본 — 다른 문서는 여기를 참조)

```
<!-- audited: YYYY-MM-DD, by: 이름, scope: core|biz-a|biz-b|biz-c -->
```

## 완료 후 (결과가 0건이어도 반드시)

1. `#agent-log`에 **`skill audit done (처리 N건: 편입 a, 수정 b, 삭제 c)`** 한 줄 기록.
   (이 기록이 routing-policy 전환 조건 "주간 심사 수행"의 유일한 증적이다)
2. `git push` 로 마무리 (스킬별 커밋은 이미 했으므로 push만).
