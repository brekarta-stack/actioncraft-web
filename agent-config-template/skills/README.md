# skills/ — 지능 계층 (심사된 지식만)

> 스킬 = 에이전트가 필요할 때만 로드하는 업무 매뉴얼.
> **원칙: 자동 생성 스킬은 자산이자 부채다. 심사 전에는 부채로 취급한다.**

## 네임스페이스

| 디렉토리 | 용도 | 편입 조건 |
|---|---|---|
| `_quarantine/` | 헤르메스가 자동 생성한 스킬의 격리 구역 | (헤르메스의 스킬 저장 경로를 여기로 설정) |
| `core/` | 전 사업 공용 (보고 형식, 로그 규칙, 산출물 생성 등) | AUDIT.md 심사 통과 + 커밋 |
| `biz-a-actioncraft/` | 사업 A 전용 (견적 응대, 웹 운영) | AUDIT.md 심사 통과 + 커밋 |
| `biz-b/` | 사업 B 전용 | AUDIT.md 심사 통과 + 커밋 |
| `biz-c/` | 사업 C 전용 | AUDIT.md 심사 통과 + 커밋 |

## 규칙

1. **에이전트는 `_quarantine/`의 스킬을 읽지도 사용하지도 않는다** (CLAUDE.md 행동 규칙 6).
2. `_quarantine/` → 다른 디렉토리로 옮기는 **유일한 방법은 주간 심사(AUDIT.md) 후 git 커밋**이다.
3. 한 스킬은 한 네임스페이스에만 속한다. 두 사업에서 쓰이면 `core/`로 승격한다.
4. 스킬 파일 상단에 심사 기록을 남긴다:
   ```
   <!-- audited: 2026-07-20, by: 대표, scope: biz-a -->
   ```
5. 90일간 사용되지 않은 스킬은 월간 점검에서 삭제를 검토한다 (부패 방지).

## 왜 이렇게 하는가

- 자동 생성 스킬의 무심사 누적은 이 시스템 오작동의 숨은 원인이었다 (아키텍처 문서 §13.2).
- git이 심사 이력을 보존하므로, 오염이 발견되면 `git log skills/` 로 범인 커밋을 찾아 1분 내 롤백한다.
