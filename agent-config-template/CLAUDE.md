# 상시 맥락 (Always-On Context)

> 에이전트가 모든 작업 전에 읽는 기본 지침. 짧게 유지한다 — 상세 지식은 skills/로.

## 회사

- 회사: [회사명] — 15인 스타트업, 3개 사업 운영
- 대표: [이름] (최종 의사결정자)
- 목표: 소수 인원 + 에이전트 체제로 운영 효율화

## 사업

| 코드 | 사업 | 핵심 업무 | 전용 채널 (PHASE 3에서 활성화 — 그 전엔 #agent로 일원화) |
|---|---|---|---|
| biz-a | 액션크래프트 / 페이퍼크래프트 (제조·웹) | 견적 문의 접수·1차 응대, 포트폴리오·블로그(CMS) 운영, 어드민 처리 | #biz-페이퍼크래프트사업 |
| biz-b | ⟪사업 B 이름 — 확인 필요⟫ | ⟪핵심 업무 1~3개 — 확인 필요⟫ (후보 채널 관측됨: #sns-biz = SNS/마케팅 카피 일일 검토) | (PHASE 3에서 #biz-b-ops 신설) |
| biz-c | ⟪사업 C 이름 — 확인 필요⟫ | ⟪핵심 업무 1~3개 — 확인 필요⟫ | (PHASE 3에서 #biz-c-ops 신설) |

> ⟪…⟫ 표시는 대표 확인이 필요한 자리입니다. 이름과 핵심 업무를 알려주면 이 표와 `skills/biz-b/`·`skills/biz-c/` 네임스페이스가 확정됩니다.
> 사업 격리 원칙상 각 사업의 고객·말투·템플릿은 해당 네임스페이스 안에서만 사용됩니다.

## 행동 규칙

1. **답변 언어**: 한국어. 대외 문서는 존댓말, 내부 로그는 간결체.
2. **에스컬레이션 — 다음은 반드시 사람 확인 후 실행**:
   - 대외 발송 (고객 이메일·견적서·계약 관련 전부)
   - 금액이 들어가는 모든 결정
   - 파일 삭제·덮어쓰기
3. **자동 실행 가능**: 분류, 초안 작성, #agent-log 보고. (내부 기록 도구는 현 단계에서
   허용된 커넥터만 — `docs/observability.md`의 MCP 커넥터 인벤토리가 원본)
4. **모든 작업 결과는 #agent-log에 한 줄 남긴다** (무엇을, 왜, 결과).
5. 확신이 없으면 추측으로 실행하지 말고 #agent-log에 질문을 남긴다.
6. **`skills/_quarantine/` 안의 스킬은 읽지도, 사용하지도 않는다.** (심사 전 격리 구역)

## 사업 간 격리

- biz-a 작업 중에는 `core/`와 `biz-a-actioncraft/` 스킬만 사용한다 (biz-b/c도 동일 원칙). 다른 사업의 고객 정보·말투·템플릿을 섞지 않는다.
